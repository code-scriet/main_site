# Python Workshop Interactive Website - Implementation Plan

## Overview
Build a standalone, multi-page interactive website for a 4-day Python workshop targeting complete beginners. Replaces traditional slides with an immersive learning platform featuring in-browser code execution, quizzes, challenges, and progress tracking.

## Tech Stack
- **Framework**: Vue.js 3 + Vite + TypeScript (Composition API)
- **Styling**: TailwindCSS (dark mode)
- **Code Execution**: Pyodide (WebAssembly Python in browser)
- **State**: Pinia + localStorage persistence
- **Routing**: Vue Router
- **Hosting**: Render (static site with SPA fallback)

## 4-Day Curriculum Structure

### Day 1: Foundations (6 sections)
1. **Welcome & First Program** - Why Python, Hello World, print()
2. **Variables & Memory** - Variables as boxes, naming conventions, reassignment
3. **Data Types** - int, float, str, bool, type(), type conversion
4. **String Operations** - Immutability, indexing, slicing, methods, f-strings
5. **Operators** - Arithmetic, comparison, logical, precedence
6. **Input/Output** - input() always returns string, formatting

### Day 2: Control Flow (5 sections)
1. **Boolean Logic** - Truthy/falsy, comparison chaining
2. **Conditionals** - if/elif/else, nested conditions, ternary
3. **While Loops** - Loop anatomy, infinite loops, break/continue
4. **For Loops** - range(), iterating sequences, enumerate()
5. **Nested Loops & Patterns** - Performance, grid problems

### Day 3: Functions & Lists (5 sections)
1. **Functions Basics** - Why functions, parameters, return, scope
2. **Advanced Functions** - *args, **kwargs, lambda, recursion
3. **Lists Fundamentals** - Creation, indexing, methods, comprehensions
4. **List Patterns** - Searching, nested lists, common operations
5. **Tuples** - Immutability, packing/unpacking, when to use

### Day 4: Data Structures & OOP (5 sections)
1. **Dictionaries** - Key-value pairs, methods, iteration
2. **Dictionary Patterns** - Nested dicts, defaultdict pattern
3. **Sets** - Unique collections, set operations
4. **File Handling** - open, read, write, context managers
5. **OOP Introduction** - Classes, objects, __init__, methods

## Key Features

### 1. Interactive Code Playground
- Monaco Editor (or CodeMirror 6) for syntax highlighting
- Pyodide lazy-loaded on first code execution
- Run button, output console, error display
- Reset to default code button

### 2. Quiz System
- **MCQ**: Single correct answer with explanation
- **Predict Output**: Show code, select expected output
- **Fill in Blank**: Complete code snippets
- Immediate feedback, retry allowed, scores tracked

### 3. Coding Challenges
- Problem statement with examples
- Test case validation (visible + hidden)
- Progressive hints system
- Solution reveal after attempts/completion

### 4. Progress Tracking (localStorage)
- Section completion checkmarks
- Quiz scores per section
- Challenge completion status
- Overall progress percentage
- "Continue where you left off" feature

### 5. Cheatsheet
- Categorized quick reference (Syntax, Operators, Data Types, etc.)
- Searchable/filterable
- Copy code snippets

### 6. Resources Page
- Official docs links
- Practice platforms (LeetCode, HackerRank)
- Video tutorials
- Recommended books

## Content Types (TypeScript)

```typescript
// Section content block types
type ContentBlock = 
  | { type: 'heading'; level: 1|2|3; text: string }
  | { type: 'text'; content: string }
  | { type: 'code'; code: string; runnable?: boolean }
  | { type: 'callout'; variant: 'info'|'warning'|'tip'; content: string }
  | { type: 'list'; items: string[]; ordered?: boolean }
  | { type: 'playground'; defaultCode: string; instructions: string }
  | { type: 'quiz'; quiz: Quiz }
  | { type: 'challenge'; challenge: Challenge }

// Quiz structure
interface Quiz {
  questions: (MCQ | PredictOutput | FillBlank)[]
}

// Challenge structure  
interface Challenge {
  title: string
  description: string
  starterCode: string
  testCases: { input?: string; expected: string; hidden: boolean }[]
  hints: string[]
  solution: string
}
```
